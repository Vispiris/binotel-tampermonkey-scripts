Стандартні конфігураційні файли

Цей документ — короткий довідник для інженерів по типових конфігураційних файлах. Його задача — швидко підказати, який конфіг брати, для чого він потрібен, що в ньому змінювати та що перевірити після налаштування.

Важливо: нестандартні конфіги додаємо тільки після погодження з супервайзером.

Швидка мапа конфігів

Як користуватись цією мапою: спочатку обери тип задачі, потім відкрий відповідний розділ нижче.

1. 

## Голосове меню / IVR


- [Звичайний IVR](#звичаи-нии-ivr) — базове меню з кнопками: 1, 2, 3. Брати, коли треба просто направити клієнта на різні сценарії.
- [IVR по часу](#ivr-по-часу) — маршрутизація за графіком. Брати, коли сценарій залежить від робочого часу, вихідних або окремих дат.
- IVR з повторами — меню повторюється кілька разів. Брати, коли клієнту треба дати 2–3 спроби вибору.
- IVR з префіксами — додає позначку до дзвінка. Брати, коли оператор має бачити, яку кнопку натиснув клієнт.
- Відображення етапу IVR при скиданні — показує, на якому пункті меню клієнт поклав слухавку. Брати для аналізу проблемних місць у меню.

2. Маршрутизація та додаткова логіка
- Sticky / Стіки — повторний дзвінок направляється на конкретну лінію або сценарій. Брати для логіки «клієнт → відповідальний співробітник».
- Донабір — клієнт сам набирає внутрішню лінію. Брати, коли треба дати можливість напряму додзвонитися до співробітника.
- Запам’ятовування вибору мови — система пам’ятає мову клієнта. Брати для української/російської або іншої мовної логіки.
- Білий список — окремі номери або пул номерів ідуть на спеціальний сценарій. Брати для VIP, операторських пулів або винятків.
- Зміна мови системних повідомлень у черзі на UA — примусово задає українську мову системних повідомлень у черзі.

3. Callback
- CallBack звичайний — дзвінок скидається, система передзвонює клієнту з внутрішньої лінії.
- CallBack зі сценарієм входу — після передзвону клієнт проходить повноцінний сценарій входу.

4. Внутрішні дзвінки та зв’язок між компаніями
- Фіксація внутрішніх дзвінків — потрібна, щоб бачити дзвінки між внутрішніми лініями в історії/статистиці.
- Дзвінки між різними договорами — потрібні для дзвінків між компаніями. Перед налаштуванням обов’язково перевірити SIP-сервер і нумерацію ліній.

5. Нестандартні конфіги — тільки після погодження
- DTMF-фіксація — фіксує натискання кнопок IVR в історії дзвінків.
- Випадковий вибір сценарію — випадково направляє дзвінок на один із кількох сценаріїв.
- Дзвінок з номера Unknown на 3Моб — специфічний кейс для дзвінків із CallerID Unknown.


Базові поняття

Що таке конфігураційний файл?
Конфігураційний файл описує логіку обробки дзвінка: яку аудіофразу програти, скільки чекати вибору клієнта, яку кнопку обробляти та на який сценарій направити дзвінок.

Що таке IVR?
IVR — це голосове меню. Наприклад: «Для відділу продажів натисніть 1, для техпідтримки — 2». У конфігу прописується, що робити після кожного натискання.

Що найчастіше змінює інженер?
- ID компанії в назві конфіга, наприклад [ivr-25809].
- ID сценаріїв, куди треба направити дзвінок: ivrRouteID=188677.
- Час очікування натискання: WaitExten(5).
- Внутрішні лінії або пул номерів: _7XX, _9XX, SIP/логіни.
- Додаткові параметри: мова, префікс, кількість повторів, графік роботи.

Сценарій за замовчуванням
Сценарій за замовчуванням — це маршрут, куди піде дзвінок, якщо клієнт нічого не натиснув або натиснув некоректну кнопку. У конфігах це часто позначено як t-сценарій.

Чек-лист перед застосуванням конфіга
- Перевірити ID компанії.
- Перевірити ID всіх сценаріїв.
- Перевірити сценарій за замовчуванням.
- Перевірити час очікування кнопки.
- Перевірити внутрішні лінії, пул номерів або SIP-логіни.
- Після додавання зробити тестовий дзвінок.

Пояснення кольорів у прикладах конфігів
🟨 Жовтий — значення, які інженер змінює під клієнта: ID компанії, ivrRouteID, номери, назви аудіо або сценаріїв.
🟦 Блакитний — логіка, яку треба уважно перевірити: час, умови, повтори, fallback або мова.
🟥 Рожевий — нестандартні / ризикові місця: міняти тільки якщо розумієш задачу або після погодження.

Стандартні конфігураційні файли

1. [[anchor:звичаи-нии-ivr]]Звичайний IVR

Для чого:
Просте голосове меню, де кожна кнопка переводить клієнта на окремий сценарій.

Коли використовувати:
Коли клієнту потрібно стандартне меню: продажі, підтримка, бухгалтерія тощо.

Що змінюємо:
- ID компанії в [ivr-25809].
- WaitExten(5) — час очікування вибору кнопки.
- ivrRouteID — ID сценаріїв для кнопок 1, 2 та t.

Приклад:
[ivr-25809]
exten => s,1,Answer(500)
exten => s,n,BackGround(${ARG1})
exten => s,n,WaitExten(5)
exten => 1,1,Set(ivrRouteID=188677)
exten => 1,n,Return
exten => 2,1,Set(ivrRouteID=188678)
exten => 2,n,Return
exten => t,1,Set(ivrRouteID=188679)
exten => t,n,Return
exten => i,1,Goto(t,1)
exten => h,1,Goto(vOfficeIvrAddHangupedCall,s,1)

Після налаштування перевірити:
- кнопка 1 веде на правильний сценарій;
- кнопка 2 веде на правильний сценарій;
- якщо нічого не натиснути, спрацьовує t-сценарій.

2. [[anchor:ivr-по-часу]]IVR по часу



### Для чого



```text
[ivr-00000]
exten => s,1,Answer(500)
```



### Підрозділ





:::important
> Важливо: коротко опиши критичний нюанс.
:::



:
Направляє дзвінок на різні сценарії залежно від часу, дня тижня або дати.

Коли використовувати:
Коли у клієнта є робочий/неробочий час, вихідні або окремі святкові правила.

Що змінюємо:
- ID компанії.
- Часові проміжки в ExecIfTime.
- ID сценаріїв для кожного періоду.
- Резервний сценарій, якщо жодне правило не підійшло.

Приклад:
[ivr-4710]
exten => s,1,Answer(500)
exten => s,n,BackGround(${ARG1},m)
exten => s,n,WaitExten(2)
exten => 0,1,Set(ivrRouteID=)
exten => 0,n,ExecIfTime(09:00-20:59,mon-fri,*,*?Set(ivrRouteID=33767))
exten => 0,n,ExecIfTime(09:00-17:59,sat-sun,*,*?Set(ivrRouteID=33767))
exten => 0,n,ExecIf($["${ivrRouteID}" == ""]?Set(ivrRouteID=38590))
exten => 0,n,Return
exten => t,1,Set(ivrRouteID=22682)
exten => t,n,Return
exten => i,1,Goto(t,1)
exten => h,1,Goto(vOfficeIvrAddHangupedCall,s,1)

Важливо:
Якщо часові правила перетинаються, спрацює правило, яке знаходиться нижче. Тому загальні правила ставимо вище, а точніші — нижче.

Приклад логіки:
exten => 0,n,ExecIfTime(09:00-20:59,mon-fri,*,*?Set(ivrRouteID=11111))
exten => 0,n,ExecIfTime(09:00-17:59,mon-mon,22-22,may?Set(ivrRouteID=22222))
exten => 0,n,ExecIf($["${ivrRouteID}" == ""]?Set(ivrRouteID=38590))

Після налаштування перевірити:
- дзвінок у робочий час;
- дзвінок у неробочий час;
- окремий день/виняток, якщо він є.

3. IVR з повторами

Для чого:
Повторює голосове меню кілька разів, якщо клієнт нічого не натиснув.

Коли використовувати:
Коли клієнту треба дати 2–3 спроби вибору перед переходом на сценарій за замовчуванням.

Що змінюємо:
- ID компанії.
- Кількість повторів у GotoIf.
- ID сценаріїв для кнопок і фінального t-сценарію.

Як працюють повтори:
- repeat=0 — на старті створюємо лічильник повторів і ставимо його в нуль.
- Коли клієнт нічого не натиснув, спрацьовує t-сценарій: repeat збільшується на 1.
- У GotoIf число 3.000000 — це ліміт повторів. Якщо треба 2 повтори — ставимо 2.000000, якщо 4 — 4.000000.
- Частина ?s,3 означає: повернутися на 3-й крок сценарію, де знову програється меню. Її міняємо тільки якщо змінилась структура рядків у конфігу.

Приклад:
[ivr-18737]
exten => s,1,Answer(500)
exten => s,n,Set(repeat=0)
exten => s,n,BackGround(${ARG1},m)
exten => s,n,WaitExten(5)
exten => 1,1,Set(ivrRouteID=189988)
exten => 1,n,Return
exten => t,1,Set(repeat=${MATH(${repeat}+1)})
exten => t,n,GotoIf($["${repeat}" != "3.000000"]?s,3)
exten => t,n,Set(ivrRouteID=117680)
exten => t,n,Return
exten => i,1,Goto(t,1)
exten => h,1,Goto(vOfficeIvrAddHangupedCall,s,1)

Нюанс:
Рядок GotoIf($["${repeat}" != "3.000000"]?s,3) означає, що IVR повториться до трьох разів. Після третьої спроби дзвінок піде на фінальний t-сценарій.

4. IVR з префіксами

Для чого:
Додає префікс до імені або номера дзвінка, щоб оператор бачив, яку кнопку натиснув клієнт.

Коли використовувати:
Коли декілька кнопок ведуть на один сценарій, але оператору важливо бачити вибір клієнта.

Що змінюємо:
- Префікси: Y, F, B або інші погоджені позначки.
- ivrRouteID для кожної кнопки.
- CALLERID(name) або CALLERID(num), залежно де треба показувати вибір.

Приклад:
[ivr-865-pokrova]
exten => s,1,Answer(500)
exten => s,n,BackGround(${ARG1})
exten => s,n,WaitExten(2)
exten => 1,1,Set(CALLERID(name)=Y ${CALLERID(name)})
exten => 1,n,Set(ivrRouteID=3671)
exten => 1,n,Return
exten => 2,1,Set(CALLERID(name)=F ${CALLERID(name)})
exten => 2,n,Set(ivrRouteID=3671)
exten => 2,n,Return
exten => 3,1,Set(CALLERID(name)=B ${CALLERID(name)})
exten => 3,n,Set(ivrRouteID=3671)
exten => 3,n,Return
exten => t,1,Set(ivrRouteID=3670)
exten => t,n,Return
exten => i,1,Goto(t,1)
exten => h,1,Goto(vOfficeIvrAddHangupedCall,s,1)

Пояснення:
CALLERID(num) — для відображення вибору в плагіні.
CALLERID(name) — для відображення вибору на програмному або IP-телефоні.

5. Sticky / Стіки

Для чого:
Реалізує липкість — повторний дзвінок клієнта направляється на конкретну внутрішню лінію або сценарій.

Коли використовувати:
Коли клієнт має повторно потрапляти до того самого співробітника або в потрібну гілку липкості.

Що змінюємо:
- ID компанії.
- Номер внутрішньої лінії в назві конфіга.
- ivrRouteID, куди має йти дзвінок.
- За потреби додаємо буквений префікс a/b/c для різних типів липкості.

Приклад:
[ivr-15535-sticky-902-]
exten => s,1,Set(ivrRouteID=193102)
exten => s,n,Return

Нюанс:
902 — внутрішня лінія, для якої створюється sticky-конфіг.

Якщо в одному сценарії використовується кілька типів липкості, робимо окремі конфіги з префіксом:
[ivr-15535-sticky-902-a]
exten => s,1,Set(ivrRouteID=193102)
exten => s,n,Return

[ivr-15535-sticky-902-]
exten => s,1,Set(ivrRouteID=222222)
exten => s,n,Return

6. Донабір

Для чого:
Дозволяє клієнту набрати внутрішню лінію при дзвінку на зовнішній номер компанії.

Коли використовувати:
Коли клієнт має можливість одразу набрати внутрішній номер співробітника.

Що змінюємо:
- ID компанії.
- Пул внутрішніх ліній, наприклад _7XX.
- t-сценарій або логіку, якщо лінію не знайдено.

Приклад:
[ivr-367-donabor]
exten => s,1,Answer(500)
exten => s,n,BackGround(${ARG1})
exten => s,n,WaitExten(1)
exten => _9XX,1,AGI(ivr.php,extNumber2extHash,${EXTEN},${groupID})
exten => _9XX,n,GotoIf($["${extHash}" = ""]?i,1)
exten => _9XX,n,Set(ivrRouteID=${extHash})
exten => _9XX,n,Return
exten => t,1,Return
exten => i,1,Goto(t,1)
exten => h,1,Goto(vOfficeIvrAddHangupedCall,s,1)

Нюанс:
_9XX змінюємо під нумерацію внутрішніх ліній клієнта.

7. Запам’ятовування вибору мови

Для чого:
Запам’ятовує, яку мову вибрав клієнт, і при наступних дзвінках направляє його у відповідну мовну гілку.

Коли використовувати:
Коли в клієнта є українська/російська або інші мовні гілки, і клієнт не має щоразу обирати мову заново.

Що змінюємо:
- ID компанії.
- ID сценаріїв для UA/RU/без вибору.
- Значення мови: ua, ru.

Приклад перевірки збереженої мови:
[ivr-53812-first-check-language]
exten => s,1,AGI(agi://127.0.0.1:3000/pbx/languageInIvr,get)
exten => s,n,BackGround(${ARG1},m)
exten => s,n,GotoIf($["${language_in_ivr}" != ""]?useLanguage)
exten => s,n,Set(ivrRouteID=283312)
exten => s,n,Return
exten => s,n(useLanguage),ExecIf($["${language_in_ivr}" != ""]?Set(CHANNEL(language)=${language_in_ivr}))
exten => s,n,ExecIf($["${language_in_ivr}" == "ua"]?Set(ivrRouteID=283310))
exten => s,n,ExecIf($["${language_in_ivr}" == "ru"]?Set(ivrRouteID=283309))
exten => s,n,ExecIf($["${ivrRouteID}" == ""]?Set(ivrRouteID=283312))
exten => s,n,Return

Приклад запису вибору мови:
[ivr-53812-mob]
exten => s,1,Answer(500)
exten => s,n,BackGround(${ARG1},m)
exten => s,n,WaitExten(5)
exten => 1,1,Set(ivrRouteID=283310)
exten => 1,n,AGI(agi://127.0.0.1:3000/pbx/languageInIvr,set,ua)
exten => 1,n,Set(CHANNEL(language)=ua)
exten => 1,n,Return
exten => 2,1,Set(ivrRouteID=283309)
exten => 2,n,AGI(agi://127.0.0.1:3000/pbx/languageInIvr,set,ru)
exten => 2,n,Set(CHANNEL(language)=ru)
exten => 2,n,Return
exten => t,1,Set(ivrRouteID=283312)
exten => t,n,Return
exten => i,1,Goto(t,1)
exten => h,1,Goto(vOfficeIvrAddHangupedCall,s,1)

8. Фіксація внутрішніх дзвінків

Для чого:
Дозволяє фіксувати дзвінки між внутрішніми лініями компанії.

Коли використовувати:
Коли клієнту потрібно бачити внутрішні дзвінки в історії або статистиці.

Що потрібно підготувати:
- Технічний номер формату 0000000000_26519 з назвою «Для фіксації внутрішніх дзвінків».
- Сценарій для фіксації внутрішніх дзвінків.
- Правило голосового меню з назвою конфіга.
- У параметрі «Здійснення дзвінків (з оновленням panel)» встановити «Користувацький».

Приклад конфігурації:

exten => _9XX,1,GotoIf($["${generalCallID}" != ""]?externalUkraine,${EXTEN},1)
exten => _9XX,n,AGI(ivr.php,extNumber2extHash,${EXTEN},${groupID})
exten => _9XX,n,GotoIf($["${extHash}" == ""]?externalUkraine,${EXTEN},1)
exten => _9XX,n,Set(extHashForRecordingInternalCalls=${extHash})
exten => _9XX,n,Goto(did2route,0000000000_26519,1)
exten => _8XX,1,Playback(vOfficeSystem/production/voice/no-route-to-destination)

include => externalUkraine

[ivr-26519-for-recording-internal-calls]
exten => s,1,GotoIf($["${extHashForRecordingInternalCalls}" == ""]?hangup)
exten => s,n,Set(ivrRouteID=${extHashForRecordingInternalCalls})
exten => s,n,Return
exten => s,n,(hangup),Playback(beep&beep)
exten => s,n,Hangup

Нюанси:
- _9XX змінюємо на пул внутрішніх ліній клієнта.
- _8XX змінюємо на пул груп виклику.
include =>  - змінюємо під локаль клієнта
9. Білий список

Для чого:
Фільтрує певні номери або пул номерів і направляє їх на окремий сценарій.

Коли використовувати:
Коли окремі номери, оператори або групи номерів мають оброблятися за іншою логікою.

Варіант 1 — фільтрація по оператору / пулу номерів:
[ivr-4564-wl]
exten => s,1,Goto(${CALLERID(num)},1)
exten => _089XXXXXXX,1,Set(ivrRouteID=158663)
exten => _089XXXXXXX,n,Return
exten => _093XXXXXXX,1,Set(ivrRouteID=158663)
exten => _093XXXXXXX,n,Return
exten => _097XXXXXXX,1,Set(ivrRouteID=158663)
exten => _097XXXXXXX,n,Return
exten => _099XXXXXXX,1,Set(ivrRouteID=158663)
exten => _099XXXXXXX,n,Return
exten => t,1,Set(ivrRouteID=125406)
exten => t,n,Return
exten => i,1,Goto(t,1)
exten => h,1,Goto(vOfficeIvrAddHangupedCall,s,1)

Варіант 2 — фільтрація по конкретному номеру:
[ivr-4564-wl]
exten => s,1,Goto(${CALLERID(num)},1)
exten => 0930021150,1,Set(ivrRouteID=158663)
exten => 0930021150,n,Return
exten => t,1,Set(ivrRouteID=125406)
exten => t,n,Return
exten => i,1,Goto(t,1)
exten => h,1,Goto(vOfficeIvrAddHangupedCall,s,1)

Після налаштування перевірити:
- номер із білого списку йде на потрібний сценарій;
- номер не з білого списку йде на t-сценарій.

10. CallBack звичайний

Для чого:
Вхідний дзвінок скидається, після чого система робить вихідний дзвінок на клієнта з однієї або кількох внутрішніх ліній.

Коли використовувати:
Коли клієнту потрібен callback без проходження сценарію входу.

Що змінюємо:
- Назву конфіга.
- SIP-логіни внутрішніх ліній.
- Пошту для фіксації необроблених callback-ів, якщо вона використовується.

Приклад:
[run-callback-14]
exten => s,1,Ringing
exten => s,n,Wait(3)
exten => s,n,Hangup(17)
exten => h,1,System(/usr/bin/php -f /var/lib/asterisk/agi-bin/tools/universal-originate.php ${sipServer} ${CALLERID(num)} "SIP/e55a887b&SIP/1e0101f2&SIP/37173ce6&SIP/80c62e19" ${groupID} "test@binotel.com" &)

Нюанс:
У параметрі "SIP/e55a887b&SIP/1e0101f2" через &SIP/ вказуються логіни внутрішніх ліній, з яких система буде виконувати callback.

11. CallBack зі сценарієм входу

Для чого:
Вхідний дзвінок скидається, система передзвонює клієнту, і після підняття слухавки клієнт проходить сценарій входу.

Коли використовувати:
Коли callback має працювати не просто як дзвінок з лінії, а як повноцінне проходження вхідного сценарію.

Що потрібно підготувати:
- Номер типу CB_номер.
- Сценарій, який має проходити клієнт після callback.
- За потреби номер, з якого має виконуватися вихідний дзвінок.

Приклад:
[run-callback-14]
exten => s,1,Ringing
exten => s,n,Wait(3)
exten => s,n,Hangup(17)
exten => h,1,System(sleep 5 && /usr/bin/php -f /var/lib/asterisk/agi-bin/tools/universal-originate-with-incoming.php ${sipServer} ${CALLERID(num)} ${didNumber} ${groupID} "test@email.com" &)

Приклад із номером для вихідного дзвінка:
[run-callback-14]
exten => s,1,Ringing
exten => s,n,Wait(3)
exten => s,n,Hangup(17)
exten => h,1,System(sleep 5 && /usr/bin/php -f /var/lib/asterisk/agi-bin/tools/universal-originate-with-incoming.php ${sipServer} ${CALLERID(num)} ${didNumber} ${groupID} "vp@binotel.ua" 0443334444 &)

Нюанс:
test@email.com або vp@binotel.ua — пошта для фіксації необроблених callback-ів. 0443334444 — номер, з якого буде виконано вихідний дзвінок.

12. Дзвінки між різними договорами

Для чого:
Дозволяє дзвонити між внутрішніми лініями різних компаній/договорів.

Коли використовувати:
Коли дві компанії мають взаємодіяти між внутрішніми лініями.

Важливо:
Перед налаштуванням перевірити, чи компанії знаходяться на одному SIP-сервері. Якщо компанії на одному SIP-сервері, конфіг додається в обидві компанії. Нумерація внутрішніх ліній не має перетинатися.

Приклад для одного SIP-сервера:
exten => _1XXX,1,AGI(ivr.php,extNumber2extHash,${EXTEN:1},6795)
exten => _1XXX,n,Dial(SIP/${extHash},30)
include => externalUkraine

Пояснення:
- 6795 — Company ID.
- _1XXX — нумерація внутрішніх ліній компанії 6795.

Якщо компанії на різних SIP-серверах:
exten => 36809,1,Goto(vOfficeCallFromCompanyToCompany,${EXTEN},1)
include => externalUkraine

Додатково:
У компанії, яка приймає дзвінок, додаємо номер 0000000000_36809 і призначаємо сценарій входу.

13. Відображення етапу IVR при скиданні

Для чого:
Дозволяє бачити, на якому етапі IVR клієнт поклав слухавку.

Коли використовувати:
Коли потрібно аналізувати, де клієнти припиняють проходження голосового меню.

Що змінюємо:
- ID компанії.
- Назви етапів у Gosub(vOfficeIvrSavePath...).
- ID сценаріїв для кожної кнопки.
- Кількість повторів, якщо вона використовується.

Фрагмент прикладу:
[ivr-18737-city-menu-ukr]
exten => s,1,Answer(500)
exten => s,n,Set(povtori=0)
exten => s,n,BackGround(${ARG1})
exten => s,n,WaitExten(5)
exten => 1,1,Set(ivrRouteID=190086)
exten => 1,n,Gosub(vOfficeIvrSavePath,s,1("Дніпро"))
exten => 1,n,Return
exten => 2,1,Set(ivrRouteID=190032)
exten => 2,n,Gosub(vOfficeIvrSavePath,s,1("Павлоград"))
exten => 2,n,Return
exten => t,1,Set(povtori=${MATH(${povtori}+1)})
exten => t,n,GotoIf($["${povtori}" != "3.000000"]?s,3)
exten => t,n,Set(ivrRouteID=117680)
exten => t,n,Return
exten => i,1,Goto(t,1)
exten => h,1,Goto(vOfficeIvrAddHangupedCall,s,1)

Нюанс:
Назви в Gosub мають бути зрозумілими, бо саме вони допомагають потім аналізувати, на якому пункті меню клієнт завершив дзвінок.

Нестандартні конфігураційні файли

Ці конфіги додаємо тільки після погодження з супервайзером. Якщо є сумнів — краще не додавати самостійно.

14. Фіксація DTMF-натискань

Для чого:
Фіксує в історії дзвінків, які кнопки IVR натискав клієнт.

Коли використовувати:
Коли клієнту потрібно бачити вибір клієнта в історії, CSV або обмежувати доступ до перегляду дзвінків за певними натисканнями.

Тип 1 — з додаванням технічного номера:
exten => 1,1,Set(ivrRouteID=283033)
exten => 1,n,AGI(agi://127.0.0.1:3000/changeDidNumber,A)
exten => 1,n,Return

Нюанс:
Для роботи потрібно додати номер, на якому призначено IVR-меню з літерною приставкою. Доступні літери: ABCDGJNPVZ.

Тип 2 — без додавання номера в компанію:
exten => 1,1,AGI(agi://127.0.0.1:3000/changeDidNumber,A,,forced)
exten => 1,n,Set(ivrRouteID=296046)
exten => 1,n,Return

Нюанс:
Історія натискань буде відображатися, але не буде можливості обмежити права доступу співробітника до перегляду дзвінків по конкретному вибору IVR.

Тип 3 — повна заміна номера для статистики:
exten => 3,1,Set(ivrRouteID=168975)
exten => 3,n,AGI(agi://127.0.0.1:3000/changeDidNumber,0441112233A,changeNumberCompletely)
exten => 3,n,Return

Коли корисно:
Якщо IVR призначений на номер з каналами типу 0441112233_1, 0441112233_2 і немає можливості додати технічний номер з літерною приставкою.

15. Випадковий вибір сценарію

Для чого:
Випадково направляє дзвінок на один зі сценаріїв.

Коли використовувати:
Коли потрібен розподіл дзвінків за випадковою логікою.

Що змінюємо:
- Діапазон RAND(1,15).
- Кількість умов ExecIf.
- ID сценаріїв.

Приклад:
[ivr-4710-rand]
exten => s,1,Set(rand=${RAND(1,15)})
exten => s,n,ExecIf($["${rand}" = "1"]?Set(ivrRouteID=23775))
exten => s,n,ExecIf($["${rand}" = "2"]?Set(ivrRouteID=23786))
exten => s,n,ExecIf($["${rand}" = "3"]?Set(ivrRouteID=23787))
exten => s,n,ExecIf($["${rand}" = "4"]?Set(ivrRouteID=23788))
exten => s,n,ExecIf($["${rand}" = "5"]?Set(ivrRouteID=23789))
exten => s,n,Return

Нюанс:
Якщо змінюється кількість сценаріїв, обов’язково змінюємо і діапазон RAND, і кількість умов ExecIf.

16. Дзвінок з номера Unknown на 3Моб

Для чого:
Дозволяє прийняти дзвінок з номера Unknown на внутрішню лінію 3Моб.

Коли використовувати:
Тільки для специфічного кейсу, коли дзвінки приходять з CallerID Unknown і їх потрібно направити на конкретний сценарій.

Що змінюємо:
- ID компанії.
- Номер, який підставляється замість Unknown.
- ivrRouteID.

Приклад:
[ivr-19446-23]
exten => s,1,Goto(${CALLERID(num)},1)
exten => Unknown,1,Set(CALLERID(num)=0441112233)
exten => Unknown,n,NoOp(${CALLERID(num)})
exten => Unknown,n,Set(ivrRouteID=298601)
exten => Unknown,n,Return
exten => t,1,Set(ivrRouteID=298601)
exten => t,n,Return
exten => i,1,Goto(t,1)
exten => h,1,Goto(vOfficeIvrAddHangupedCall,s,1)

17. Зміна мови системних повідомлень у черзі на UA

Для чого:
Примусово встановлює українську мову системних повідомлень у сценарії з чергою.

Коли використовувати:
Коли дзвінок завжди має йти на один сценарій з чергою, але системні повідомлення потрібно програвати українською.

Що змінюємо:
- ID компанії.
- ivrRouteID сценарію з чергою.
- За потреби мову в CHANNEL(language)=ua.

Приклад:
[ivr-63238-language]
exten => s,1,Answer(500)
exten => s,n,BackGround(${ARG1},m)
exten => s,n,Set(CHANNEL(language)=ua)
exten => s,n,WaitExten(1)
exten => t,1,Set(ivrRouteID=328305)
exten => t,n,Return
exten => i,1,Goto(t,1)
exten => h,1,Goto(vOfficeIvrSuccessAddHangupedCall,s,1)

Фінальна перевірка після будь-якого конфіга

Перед тим як закривати задачу, інженер має перевірити:
- тестовий дзвінок проходить за очікуваною логікою;
- всі кнопки IVR ведуть на правильні сценарії;
- сценарій за замовчуванням працює;
- у статистиці/історії дзвінок відображається коректно;
- якщо є часові правила — перевірені робочий і неробочий сценарії;
- якщо є префікси, sticky, DTMF або callback — перевірено саме їхню додаткову логіку.
